# .env Quick Reference

## Setup (One Time)

```powershell
# 1. Copy template
Copy-Item .env.example .env

# 2. Edit .env with your values
code .env

# 3. Load into session
.\Load-EnvFile.ps1
```

## Daily Usage

```powershell
# Load environment variables into your session
.\Load-EnvFile.ps1

# Or dot-source it to load into current scope
. .\Load-EnvFile.ps1

# Verify variables loaded
$env:AZURE_SUBSCRIPTION_ID
```

## Common Variables

| Variable | Description | Example |
|----------|-------------|---------|
| `AZURE_SUBSCRIPTION_ID` | Azure subscription GUID | `12345678-1234-1234-1234-123456789abc` |
| `AZURE_TENANT_ID` | Azure AD tenant GUID | `87654321-4321-4321-4321-cba987654321` |
| `AZURE_RESOURCE_GROUP` | Default resource group | `my-dev-rg` |
| `AZURE_LOCATION` | Default region | `eastus` |
| `OUTPUT_PATH` | Output directory | `./output` |
| **Dashboard Management** | | |
| `DASHBOARD_CREATOR_NAME` | **CRITICAL**: Your full name for filtering dashboards | `Your Full Name` |
| `DASHBOARD_OUTPUT_DIR` | Directory for exported dashboards | `./output/dashboards` |
| `DASHBOARD_MANIFEST_FILE` | Export manifest filename | `manifest.json` |

## Using in Scripts

```powershell
# As parameter defaults
param(
    [string]$SubscriptionId = $env:AZURE_SUBSCRIPTION_ID
)

# Direct reference
$subId = $env:AZURE_SUBSCRIPTION_ID

# With fallback
$rg = if ($env:AZURE_RESOURCE_GROUP) { 
    $env:AZURE_RESOURCE_GROUP 
} else { 
    "default-rg" 
}
```

## Troubleshooting

```powershell
# Show all environment variables starting with AZURE_
Get-ChildItem env:AZURE_*

# Reload with verbose output
.\Load-EnvFile.ps1 -Verbose

# Override existing variables
.\Load-EnvFile.ps1 -Force
```
